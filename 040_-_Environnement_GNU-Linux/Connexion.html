Connexion

Créée le lundi 28 avril 2025

Pas de connexion directement en root sur un serveur. On se connecte en utilisateur et ensuite on demande les accès root. Sécurité supplémentaire.

exit déconnexion de la session SSH

/etc/motd - Stocke le message de bienvenue lors d'une connexion

/etc/passwd : Fichier qui liste les comptes utilisateurs et les accès, le nom d'utilisateur, l'UID (identifiant unique de l'utilisateur), le GID (identifiant de groupe), le répertoire personnel, le shell par défaut, etc.

user1:x:1001:1001:Utilisateur:/home/user1:/bin/bash
x dans le champ du mot de passe signifie que le mot de passe est stocké ailleurs (dans /etc/shadow).

/etc/shadow : Ce fichier contient les informations des mots de passe chiffrés des utilisateurs. Il est lisible uniquement par l'utilisateur root pour des raisons de sécurité. Chaque ligne contient des informations sur l'utilisateur, comme son mot de passe chiffré, la date du dernier changement de mot de passe, et des informations sur les politiques de changement de mot de passe.

Sur la plupart des distributions Linux modernes, le hachage par défaut des mots de passe est basé sur l'algorithme SHA-512.
Cela signifie que les mots de passe sont hachés à l'aide de SHA-512 dans le fichier /etc/shadow en utilisant une méthode appelée "salage" (salt), qui ajoute un ensemble aléatoire de caractères au mot de passe avant de le hacher. Cela rend les attaques par force brute beaucoup plus difficiles.
